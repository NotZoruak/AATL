# AATL 开发日志

## v0.1 — 远征自动化（2026-06-08）

### 新增
- 远征任务全流程自动化：本丸检测 → 队伍状态检查 → 空闲队伍派遣 → GoHome 回城 → 循环
- 三路并行入口检测：远征归来收取、远征界面脱困、本丸主页识别
- 部队一~五独立配置远征地图（20 张可选 + 休息），休息队伍自动跳过
- 可自定义刷新间隔（每次派遣后的等待时间）
- 每次点击加入随机位置偏移，模拟真实操作
- 启动时自动生成流水线坐标，修改配置后重启即生效

### GUI
- 基于 MFAAvalonia v2.12.1 源码构建，项目名为 AATL.exe
- 自定义程序图标
- 最小化到系统托盘
- 设置面板：5 个部队下拉框 + 刷新间隔输入框
- 所有节点失败时自动回退到起始循环，不会卡死

### 流水线
- 迁移至嵌套格式（MaaFramework 新版）
- 全局速率限制 + 点击前后延迟，适配模拟器性能
- 远征归来使用 OCR 快速检测（不平级等待）
- GoHome 链路：目录 → 验证 → 回本丸 → 等刷新 → 重新循环

### 项目结构
- 扁平化根目录，双击 AATL.exe 即用
- 移除 MFAToolsPlus（独立工具，不随项目分发）
- 移除 GUI/ 子文件夹
- 新增 default_pipeline.json 全局配置

### 修复
- `next` 数组语义：引擎依次尝试列表中的节点，首个命中即执行
- 批量 OCR 导致队伍检查循环的问题，改用 OR 列表 + enabled=false 跳过休息队伍
- TemplateMatch 无法设置节点超时的问题，改用 OCR 快速判断
- GoHome 后任务结束的问题，改为循环回到 Expedition 入口

## v0.3.1 — 合战场通用流水线（2026-06-10）

### 新增
- Sortie.json 通用合战场流水线（100+ 节点）：支持自选时代/地域/阵形/部队的参数化出阵
- 双轮主枢纽 + 行军后枢纽 + 即刻出阵后枢纽四枢纽架构，避免动画帧任务终止
- 王点前撤退功能：四图模板匹配 + 阈值 [0.98]，行军前先检测再决定点不点
- 重伤检测双时机（出阵前/行军中），修刀全链路：目录→修复→选槽→加速符循环→确认→出阵
- 重伤处理下拉框（修刀/停止）互斥二选一
- 加速符匹配循环：反复点未使用直到匹配已使用

### GUI
- 勾选框改为内嵌文字 + 每行两个 UniformGrid 布局
- 阵形下拉框新增"不启用"选项
- 江户城模板匹配阈值统一提升至 0.98
- 任务描述更新：补充刀装与同步远征标注开发中

### 修复
- 主枢纽 IsMarching 之前直接点击行军绕过王点检测，改为路由到 CheckBossBeforeMarch
- TemplateMatch threshold 必须用数组 `[0.98]` 格式，单值会被忽略
- 即刻出阵后枢纽缺少阵形检测和兜底导致首战终止
- 重伤检测盲点提前关闭弹窗导致 OCR "否" 失效
- 修复链多处 OCR 换盲点 + 延迟解决菜单动画时序问题
- 时代选择漏加右下角确认点击，导致卡在地域选择
- 阵形双击间隔从 500ms 降至 200ms，兜底从 1.5s 降至 1s

### 性能优化（2026-06-10）
- 大量识别节点从 OCR 改为模板匹配（远征结果、行军、战斗结果、阵形选择、时代选择、地域选择、出阵活动、本丸等），消除 OCR 延迟与误判
- 所有模板匹配统一启用 green_mask + threshold [0.98]
- 所有枢纽节点增加 pre_delay: 0，消除不必要等待

### 菜单流程优化

- 点击菜单按钮后移除目录验证直接走枢纽，避免时代选择界面误匹配导致 20 秒死循环
- 四项流水线（远征/江户城/刷花/合战场）目录验证全部统一为模板匹配
- ClickSortieInMenu / ClickRepairInMenu 新增 on_error 兜底，找不到目标立即回枢纽

### 合战场部队选择修复
