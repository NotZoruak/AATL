# AATL 开发日志

## v0.1 — 远征自动化（2026-06-08）

### 新增
- 远征任务全流程自动化：本丸检测 → 队伍状态检查 → 空闲队伍派遣 → GoHome 回城 → 循环
- 三路并行入口检测：远征归来收取、远征界面脱困、本丸主页识别
- 部队一~五独立配置远征地图（20 张可选 + 休息），休息队伍自动跳过
- 可自定义刷新间隔（每次派遣后的等待时间）
- 每次点击加入随机位置偏移，模拟真实操作
- 启动时自动生成流水线坐标，修改配置后重启即生效

### GUI
- 基于 MFAAvalonia v2.12.1 源码构建，项目名为 AATL.exe
- 自定义程序图标
- 最小化到系统托盘
- 设置面板：5 个部队下拉框 + 刷新间隔输入框
- 所有节点失败时自动回退到起始循环，不会卡死

### 流水线
- 迁移至嵌套格式（MaaFramework 新版）
- 全局速率限制 + 点击前后延迟，适配模拟器性能
- 远征归来使用 OCR 快速检测（不平级等待）
- GoHome 链路：目录 → 验证 → 回本丸 → 等刷新 → 重新循环

### 项目结构
- 扁平化根目录，双击 AATL.exe 即用
- 移除 MFAToolsPlus（独立工具，不随项目分发）
- 移除 GUI/ 子文件夹
- 新增 default_pipeline.json 全局配置

### 修复
- `next` 数组语义：引擎依次尝试列表中的节点，首个命中即执行
- 批量 OCR 导致队伍检查循环的问题，改用 OR 列表 + enabled=false 跳过休息队伍
- TemplateMatch 无法设置节点超时的问题，改用 OCR 快速判断
- GoHome 后任务结束的问题，改为循环回到 Expedition 入口
